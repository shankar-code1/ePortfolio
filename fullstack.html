What is full stack development?

“Full stack means working on both the front end and the back end of a website or web application. The front end is the part users see and interact with, like buttons, pages, and layouts. It is usually built using technologies such as HTML, CSS, and JavaScript. The back end is the part behind the scenes. It handles things like servers, databases, and application logic. Technologies such as Node.js, Python, and databases like MySQL are commonly used. A full stack developer is someone who can work on both sides, meaning they can build what users see and also manage the system that runs in the background.”

1. Frontend in action

“Frontend is the part of a website or web application that users see and interact with directly. When you open a website, everything on the screen—like the layout, buttons, images, menus, and text—is part of the frontend. In simple words, the frontend focuses on how the website looks and feels. It makes sure the page is easy to use, visually attractive, and works smoothly when someone clicks or types something. Frontend developers usually build this part using technologies such as HTML to create the structure of the page, CSS to design and style it, and JavaScript to add interactivity like animations, forms, and dynamic content.”

Below is a minimal but complete frontend interaction built with HTML, CSS (via Bootstrap), and vanilla JavaScript. It shows how JavaScript directly manipulates the DOM.

0

// Frontend JavaScript (DOM manipulation)
let count = 0;
const counterEl = document.getElementById('counterValue');
const incBtn = document.getElementById('incrementBtn');
const resetBtn = document.getElementById('resetBtn');

incBtn.addEventListener('click', () => {
  count++;
  counterEl.textContent = count;
});

resetBtn.addEventListener('click', () => {
  count = 0;
  counterEl.textContent = count;
});

The counter above uses exactly this code – fully interactive. That’s the essence of frontend: structure (HTML), style (CSS classes), and behaviour (JS).

🧠 Key takeaway: Frontend is what users see and touch. It's built with HTML, CSS, and JavaScript, and it can communicate with the backend to fetch or send data.

2. Backend example (PHP)

“Backend is the part of a website or web application that works behind the scenes. Users usually don’t see it, but it is what makes the website actually function. The backend handles things like storing data, processing requests, managing users, and connecting the website to a database. For example, when someone logs into a website or submits a form, the backend processes that information and sends the correct response. Backend development is usually done with programming languages such as Python, PHP, or Node.js, and it often uses databases like MySQL.”

Here is a simple backend API written in PHP that returns a list of users as JSON – exactly what a frontend would fetch to display profiles.

<?php
// api/users.php – backend endpoint written in PHP
header('Content-Type: application/json');

// Simulated data (in a real app this would come from a database)
$users = [
  ['id' => 1, 'name' => 'Shankar', 'role' => 'full stack dev'],
  ['id' => 2, 'name' => 'Sita', 'role' => 'frontend specialist'],
  ['id' => 3, 'name' => 'Rajan', 'role' => 'backend engineer']
];

// Output JSON
echo json_encode($users);
?>

If you place this file on a PHP server (like Apache or php -S localhost:8000), visiting /api/users.php returns a JSON array. A frontend can fetch it using fetch('/api/users.php').

Real‑world: A full stack developer would connect this to a MySQL database, add validation, authentication, and handle different HTTP methods.

🧠 Key takeaway: The backend runs on a server, processes logic, and talks to the database. It provides data and services to the frontend.

3. Database example (SQL)

“A database is a place where a website or application stores and organizes data so it can be easily accessed and managed later. For example, when you create an account on a website, your username, password, and other details are saved in a database. The system can then quickly find that information when you log in again. Databases help websites store large amounts of information in an organized way. Developers often use systems like MySQL or MongoDB to manage this data.”

Below is how you would define and query a table using SQL (MySQL/PostgreSQL). This is the language used to interact with relational databases.

🗃️ SQL (MySQL)

-- Create a table to store users
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Insert a new user (e.g., during registration)
INSERT INTO users (name, email) 
VALUES ('Shankar', 'shankar@fullstack.dev');

-- Retrieve all users (e.g., for an admin panel)
SELECT * FROM users;

-- Find a specific user by email (e.g., login)
SELECT * FROM users WHERE email = 'shankar@fullstack.dev';

-- Join example (if there were orders)
SELECT users.name, orders.total
FROM users
JOIN orders ON users.id = orders.user_id;

These SQL statements would be embedded in backend code (like PHP) to persist and retrieve data. Understanding SQL is essential for any full stack developer.

🧠 Key takeaway: The database stores all persistent data. It is the foundation that both the backend and frontend ultimately rely on.

Full stack = frontend + backend + database

The three layers work together: the frontend (what users see) talks to the backend (server logic), which reads from and writes to the database. As a junior full stack developer, I'm comfortable working across all three.
